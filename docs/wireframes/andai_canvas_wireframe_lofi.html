Spec Andai — meja perencanaan

  • Bukan Cek Dulu versi panjang. Cek Dulu = 1 barang, instan, "sampai gajian". Andai = banyak variabel ditumpuk, hipotetis, horizon akhir bulan + 3 bulan.
  • Baseline di atas. "Sekarang tanpa diandai" jadi titik banding — dunia nyata sebelum dicabang (sesuai icon git-fork).
  • Stack pengandaian. User numpuk beberapa "andai" sekaligus. 4 variabel terkunci: beli (pengeluaran sekali), income (pemasukan beda — naik/turun/telat/bonus), tagihan (komitmen rutin baru/berubah), target nabung (setoran goal). Di state ini: service mobil 5jt + gaji telat 1 minggu. Inilah multi-variabel yang sengaja diusir dari Cek Dulu.
  • Hasil horizon panjang. Jatah harian sampai gajian, sisa pas gajian, total tabungan, lalu forecast 3-bulan — angka polos, tanpa kata opinionated. Forecast = tetangga Andai, bukan Cek Dulu.
  • TIER — pembeda satu hal telak: persistence. Hitungannya identik Basic & Pro. Basic bisa main 1 skenario penuh; tutup canvas → ilang. Pro bisa simpan & bandingin multi skenario berdampingan.
  • Tag Pro DI DALAM, bukan di entry point. Tombol Andai di home polos (no padlock). Basic user ngerasain hitungan penuh dulu — gembok baru muncul di tombol "Simpan skenario". Momen konversi, bukan dinding di depan pintu.
  • Forecast 3-bulan juga Pro. Basic lihat hasil "akhir bulan ini" saja; blok forecast diganti teaser/terkunci.
