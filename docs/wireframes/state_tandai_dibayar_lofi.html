Spec — tandai sudah dibayar (push sheet)

  • Push sheet penuh (bukan modal bottom), karena 3 input + edge case past date = butuh ruang. Header punya tombol back, bukan close.
  • Entry point: dari notif sheet (button "tandai sudah dibayar") ATAU dari swipe-left di list komitmen. Sheet sama persis, ga tergantung asal.
  • Preview item di atas: nama, tag tipe komitmen (tetap / variabel), estimasi nominal. Tag "variabel" + prefix "± Rp" sebagai cue ke user bahwa nominal mungkin beda.
  • Auto-log note dashed border = transparansi: konfirmasi otomatis kurangi saldo wallet + kebuat di log catatan. User paham konsekuensinya sebelum tap Konfirmasi.
  • Nominal editable: pre-fill estimasi (variabel) atau nilai fix (tetap). Saat user edit, muncul hint selisih dari estimasi (case 1: "+Rp 147.500 · normal untuk komitmen variabel"). Bahasa netral, ga judgmental.
  • Wallet picker: radio list, ga collapsed. User langsung lihat saldo per wallet untuk pilih. Wajib pilih satu — ga ada default auto-select.
  • Insufficient saldo (kalau wallet saldo < nominal): muncul warning kecil di bawah picker, tapi tetep bisa dipilih (mungkin user nemu transfer-an masuk yang belum dicatat). Trust user.
  • Tanggal picker: row utama show tanggal terpilih + sub-info hari/relasi. Quick pills di bawah: "hari ini", "kemarin", "2 hari lalu", "pilih tanggal ›". Pills paling sering dipakai → langsung 1 tap.
  • "Pilih tanggal ›" buka native date picker iOS/Android — pattern modern 2026, ga reinvent custom calendar.
  • Konfirmasi → 4 hal terjadi sekaligus: (1) saldo wallet berkurang sesuai nominal, (2) entry baru di log catatan dengan tag "komitmen · nama_komitmen", (3) komitmen di-mark paid untuk siklus ini & hilang dari list aktif, (4) toast confirmation di home dengan undo.
  • Batal: tutup sheet, balik ke list. Komitmen tetap di posisi awal (belum dibayar).
