Spec — swipe-to-pay flow (3 stage)

  • Stage 1: Swipe reveal — user swipe kiri item, container item geser ke kiri ~110px, reveal action panel hitam di kanan. Icon ✓ + label "tandai dibayar". Pattern native iOS/Android, ga aneh.
  • Trigger threshold: kalau swipe >50%, swipe selesai auto otomatis → langsung buka modal. Kalau swipe <50%, item snap balik ke posisi semula.
  • Stage 2: Modal konfirmasi — bottom modal (bukan bottom sheet panjang), karena ini decision point pendek. List dim di belakang untuk konteks.
  • Modal content: nominal pre-fill (editable kalau variabel ±), wallet picker dengan note saldo, tanggal bayar (default hari ini). 3 input, ga lebih.
  • Tombol: "Batal" (ghost, kiri) + "Konfirmasi" (primary, kanan). Pattern modern 2026 — primary action di kanan untuk thumb reach.
  • Stage 3: Toast confirmation — toast hitam muncul di atas bottom action area (bukan di top, karena attention user lagi di bottom). Auto-dismiss 5 detik, progress bar tipis di bawah.
  • Undo affordance: tombol "Ubah" + "Batal" di toast. "Ubah" → buka modal lagi dengan field pre-filled. "Batal" → revert (item balik ke list, transaksi dihapus).
  • Item Spotify hilang dari list setelah confirm — tracking-wise, dia udah dibayar bulan ini, jadi ga muncul lagi sampai siklus berikutnya. Tagihan baru (Netflix) naik ke atas dari expand list.
  • Haptic: light di swipe trigger · medium di confirm tap · light di toast auto-dismiss.
