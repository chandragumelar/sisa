Spec — Quick Log Sheet (PRD §3.4.11)

Pattern bottom sheet (semua mode)

  • Bentuk: bottom sheet, bukan push penuh. Drag handle di atas, tap area dim = dismiss.
  • Prinsip: nominal > wallet > sisanya opsional. Idealnya 2 aksi inti: pilih/terima wallet default → ketik nominal → tap Catat.
  • Toggle 3 segmen di atas: keluar / masuk / nabung. Default keluar. Toggle ubah field di bawahnya — tiap mode beda isi.
  • Wallet picker selalu paling atas (di bawah toggle), karena wallet menentukan currency symbol di nominal. Default = wallet terakhir dipakai.
  • Nominal hero: angka besar, prefix currency mengikuti wallet terpilih. Numpad auto-focus.
  • Tombol Catat disabled sampai nominal terisi (state di wireframe = belum input).

Diferensiasi per mode

  • Keluar — label opsional · toggle "dari tabungan" (off default, mekanisme tarik tabungan) · tanggal · catatan.
  • Masuk — label kontekstual (gaji, freelance, transfer masuk, bonus) · tanggal · catatan. Tidak ada toggle dari tabungan.
  • Nabung — 2 skenario radio: (A) earmark di dompet sumber (saldo total tetap, muncul "terkunci goal") vs (B) pindah ke dompet nabung lain · tanggal. Tidak ada step pilih goal — paradigma celengan ayam, otomatis ke goal teratas, luber ke bawah.

Kenapa nabung jadi segmen sendiri

  • Secara mental tidak bisa dipaksa masuk "keluar" atau "masuk" — membingungkan user.
  • Tarik tabungan = mode keluar + toggle "dari tabungan" (bukan flow terpisah).
  • Satu cabang logika tunggal: tarik > total tabungan → app tanya sisanya dari mana (conversational).

Toast confirmation (post-Catat, semua mode)

  • Hitam, di atas bottom action, countdown 5 detik auto-dismiss, tombol Ubah + Batal.
  • Mode nabung: "✓ Nabung Rp X · masuk [nama goal teratas] · barusan"
  • Batal = revert atomik: transaksi terhapus + total tabungan dikembalikan + goal dihitung ulang. Tidak ada revert setengah.
